# 设计升级方案 · v0.2

> 当前版本（v0.1）功能完整，但视觉语言偏通用 SaaS 风格 —— 没有自己的
> 辨识度。这份文档规划 v0.2 的视觉与体验升级方向。

## 现状评估

✅ **做得好的**
- 信息层级清晰：sidebar / topbar / content 三段式
- 数据密度合理：4 列看板、3 列统计卡
- 暗色模式完整实现
- Markdown 渲染有专门的样式（H1 / 引用 / 代码块 / wikilink 胶囊）
- 响应式断点处理移动端

⚠️ **需要改进的**
- 配色单调：单一紫粉渐变，缺乏个性
- 字体全系统字体，无显示字体的层次感
- 人物头像都是 init，缺少视觉锚点
- 卡片视觉权重一致，没有按类型区分
- 空状态文案功能化，缺少人格
- 没有微交互（hover、点击反馈、过渡动画）
- 信息密度可更高（dashboard 右侧大片空白）

---

## 升级方向

v0.2 的视觉目标是：**「有自己性格的、warm 的、knowledge-first 的工具」**。

参考坐标（不模仿，找感觉）：
- **Linear**：密度、信息层级、操作确定性
- **Capacities**：实体类型配色、知识网络的视觉感
- **Notion**：内容优先、克制的装饰
- **Craft**：暖色系、衬线显示字、留白节奏

### 1. 配色系统：实体类型上色

每个实体类型有专属强调色，用于卡片左侧色条、链接胶囊、统计图表：

```
人物 (person)   → 暖橘 #f97316 / #fb923c
任务 (task)     → 蓝青 #0ea5e9 / #38bdf8
项目 (project)  → 紫罗兰 #8b5cf6 / #a78bfa
链接 (link)     → 翡翠绿 #10b981 / #34d399
```

**应用位置**：
- 卡片左侧 3px 竖条（type-color）
- 详情页头部横线 / 顶条
- wikilink 胶囊背景（更柔和的同色系）
- 标签云按 type 分组显示

**结果**：用户扫一眼就知道这是人物还是任务，不用读文字。

### 2. 字体升级

引入一个**显示字体（display font）**给大标题，建立视觉层级：

```css
--font-display: 'Fraunces', 'Iowan Old Style', 'Source Han Serif SC', serif;
--font-sans: 'Inter', -apple-system, 'PingFang SC', 'Microsoft YaHei', sans-serif;
--font-mono: 'JetBrains Mono', 'SF Mono', monospace;
```

具体调整：
- **大标题（h1, h2, modal-title）** → Fraunces serif
- **正文、UI** → Inter sans
- **代码、slug** → JetBrains Mono

通过 Google Fonts / Bunny Fonts CDN 加载，本地 fallback 优雅降级。

### 3. 头像生成

每个人物自动生成**程序化头像**：
- 取名字的稳定 hash 值
- 选择 12 种预设渐变之一
- 显示首字（中文或英文）
- 可选：手绘风格的简单图形叠加（后期）

效果：

```

│ 橘红   │  │ 蓝绿   │  │ 紫粉   │
│ 渐变   │  │ 渐变   │  │ 渐变   │
└────────┘  └────────┘  └────────┘
```

### 4. 卡片视觉层级

**人物卡片**：左侧 type-color 竖条 + 头像 + 名称 + 元信息
**任务卡片**：左侧 type-color 竖条 + 标题 + 优先级条 + 截止日期
**项目卡片**：左侧 type-color 竖条 + 标题 + 描述预览 + 关联数
**链接卡片**：封面图（如果有）+ link preview + 抓取元信息

卡片 hover：
- 上浮 2px
- 阴影加深
- 边框颜色变成 type-color

### 6. 看板交互升级

- **拖拽切换列**：用 HTML5 drag/drop API
- **inline 状态切换**：卡片右上角悬浮一个小状态切换器
- **列内排序**：同列内可拖拽排序
- **快速添加**：列底「+ 添加任务」直接创建
- **WIP 限制**：每列可设上限，超过显示警告色

### 7. 微交互

```css
/* 默认过渡 */
* { transition: background-color 120ms, border-color 120ms, transform 120ms; }

/* 卡片 hover */
.card:hover {
  transform: translateY(-2px);
  box-shadow: var(--shadow-md);
}

/* 按钮点击反馈 */
.btn:active { transform: scale(0.97); }

/* 模态框进出 */
.modal { animation: modal-in 180ms ease-out; }

/* wikilink hover */
.wikilink:hover { transform: scale(1.02); }

/* toast 进入 */
.toast { animation: toast-in 200ms ease-out; }
```

### 8. 空状态插画

每个空状态加**程序化插画**（纯 SVG，CSS 着色）：

- 人物空状态：简约人形剪影 + 书本/电脑
- 任务空状态：勾选框 + 虚线
- 项目空状态：文件夹 + 连接线
- 链接空状态：链条 / 网络节点

风格：扁平、几何、与品牌色一致。

### 10. 主题系统

用户可在 Settings 里选主题：

- **Light** (默认)
- **Dark**
- **Sepia** — 暖色护眼，适合长读
- **High Contrast** — 无障碍
- **Custom** — 选 accent color + 字体大小

CSS variables 驱动，整套换色不需改 JS。

### Phase 3 — 内容呈现（1 周）

## 风险

- **字体加载慢**：首次访问要下载 Fraunces / Inter → 用 `font-display: swap` + preload
- **拖拽在移动端体验差**：mobile 上改用 long-press menu
- **类型配色冲突**：橘 + 红 + 绿 放一起可能刺眼 → 调饱和度到 60-70%，避免纯色
- **审美疲劳**：设计升级是高投入低频操作，确认方向后再大改

---

## 给你过目的决策点

最终确认前需要敲定的：

1. **字体方向**：衬线（Fraunces 风格，温暖）vs 无衬线（Inter 风格，精确）
2. **类型配色**：上面建议的暖橘 / 蓝青 / 紫罗兰 / 翡翠绿 行不行
3. **Dashboard 改版**：填满空白（高密度）vs 保持留白（呼吸感）
4. **优先级**：Phase 1 → 4 你想按顺序全做，还是先做某个？
